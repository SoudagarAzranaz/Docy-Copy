Unordered Lists with an example


An unordered list in HTML is created using the <ul> element. The <ul> element is used to define an unordered list, and each list item is defined using the <li> element. The items in an unordered list are typically displayed with bullet points or dots.
Example: 

                                                   
                                                <ul> <li>Apples</li> 
<li>Bananas</li>
 <li>Oranges</li> ,
<li>Pears</li>
 </ul>


In the above example, an unordered list is defined using the <ul> element. The list items are defined using the <li> elements, and the items will be displayed with bullet points or dots. The items in this example are "Apples", "Bananas", "Oranges" and "Pears".

Note: <ul> element is used to create an unordered list, and <li> element is used to define each list item, it can be nested in other elements such as <body> or <section>.

Note: <ol> element is used to create an ordered list, and <li> element is used to define each list item, it can be nested in other elements such as <body> or <section>.

Nesting Lists with an example


Lists can be nested in HTML by placing one list inside another list element. This allows for the creation of more complex and organized lists.
Example: 

                                               
                                               
 <ul>
 <li>Fruits </li>
 <li>Apples</li> 
 <li>Bananas</li>  
<li>Oranges</li>  </ul>  
 <li>Vegetables </li> 
 <li>Carrots</li> 
 <li>Potatoes</li> 
 <li>Tomatoes</li> 
 </ul>  
</li> 
 </ul>


In the above example, an unordered list is defined using the <ul> element. The list items are defined using the <li> elements. The first-level list items are "Fruits" and "Vegetables". The second-level lists are created by nesting another <ul> element inside the <li> element of the first-level list. The second-level list items are different types of fruits and vegetables.

Note: Lists can be nested using any type of list element such as <ol> or <dl>, the nesting can be made as deep as needed to achieve the desired structure.

    Creating Accessible Links with an example

    Creating accessible links in HTML is important for making sure that users with assistive technologies (such as screen readers) can navigate and understand the content of a webpage.


    Here are some best practices for creating accessible links:



    Use descriptive link text: The text within the <a> tags should be descriptive and give context to the user about where the link will take them. Avoid using phrases like "click here" or "read more" as they do not give any context on their own. 

                                                            
                                                   
    <a href="https://www.example.com">Learn more about our company</a>



    Use the title attribute: The title attribute can be used to provide additional information about a link. This information will be read by screen readers but not visible on the page. 

                                                            
                                                    <a href="https://www.example.com" title="Opens in a new window">Learn more about our company</a>




    Avoid using the name attribute: Using the name attribute for links is considered as obsolete and it's not recommended. Instead, it's recommended to use the id attribute, because it's more semantic and it's used for other things as well such as CSS and JavaScript. 

                                                                            
                                                                <a href="#section1">Go to Section 1</a> ... <h2 id="section1">Section 1</h2>


    Use the role attribute: The role attribute can be used to provide additional information about the link's purpose. 

                                                                        
                                                                <a href="https://www.example.com" role="button">Learn more about our company</a>



    Use the tabindex attribute: The tabindex attribute can be used to set the order of the links when using the tab button. 

                                                                        
                                                               <a href="https://www.example.com" tabindex="1">Learn more about our company</a>

    It's important to note that creating accessible links is important for accessibility and it's an ongoing effort, it's a good idea to test the links with screen readers and other assistive technologies to make sure that they are working correctly and that the links are accessible to all users.
